## 2. 프로필 & 멤버 관리

### 2.1 내 프로필 (`/community/profile`)
- [ ] 프로필 정보 표시 (이름, 소속, 역할, 소개글)
- [ ] 프로필 편집 버튼 작동
- [ ] 프로필 공개/비공개 토글 작동
- [ ] 아바타 이미지 업로드/변경
- [ ] 소속 정보 수정 (회사, 직책)
- [ ] 두 번째 소속 정보 추가 가능
- [ ] 소개글 작성/수정 (3줄까지 표시)
- [ ] 보유 뱃지 표시 (최대 3개, 세로 배치)
- [ ] 뱃지 공개/비공개 설정
- [ ] 비활성화된 뱃지는 표시되지 않음
- [ ] 우측 사이드바 없음 (전체 너비 사용)
- [ ] 내 게시글/이벤트 목록 표시

### 2.2 멤버 리스트 (`/member`)
- [ ] 공개 프로필만 표시
- [ ] 3열 그리드 레이아웃 (lg 이상)
- [ ] 2열 그리드 (md)
- [ ] 1열 그리드 (모바일)
- [ ] 프로필 사진, 이름, 소속, 소개글 표시
- [ ] 뱃지 표시 (최대 3개)
- [ ] 카드 호버 시 그림자 효과
- [ ] 카드 클릭 시 상세 모달 표시
- [ ] 비공개 프로필 사용자에게 공개 유도 배너 표시
- [ ] 로딩 스켈레톤이 실제 레이아웃과 일치

### 2.3 멤버 상세 모달
- [ ] 프로필 사진, 이름
- [ ] 소속 정보 (1개 또는 2개)
- [ ] 역할 뱃지
- [ ] 보유 뱃지 (활성화된 것만)
- [ ] 소개글
- [ ] 닫기 버튼 작동
- [ ] Master 권한: 리더 지정 버튼 없음 (커뮤니티별로 이동)

## 점검 방법

### 1. 로컬 개발 환경에서 테스트
```bash
npm run dev
```
- 각 페이지를 직접 방문하여 기능 테스트
- 브라우저 콘솔에서 에러 확인
- Network 탭에서 API 요청 확인

### 2. 다양한 계정으로 테스트
- 비로그인 사용자
- 일반 회원 (member)
- 커뮤니티 리더
- Admin
- Master

### 3. 다양한 디바이스/브라우저
- Desktop (Chrome, Safari, Firefox)
- Tablet
- Mobile (iOS Safari, Android Chrome)

### 4. 자동화 테스트 (추후)
```bash
# E2E 테스트 (Playwright 등)
npm run test:e2e

# 컴포넌트 테스트 (Jest + React Testing Library)
npm run test:unit
```
